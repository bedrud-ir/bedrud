## Task 15: iOS — Fix `RoomDelegateHandler` `@unchecked Sendable` weak var access

**Files:**
- Modify: `apps/ios/Bedrud/Core/LiveKit/RoomManager.swift` (`RoomDelegateHandler`, ~line 377)

- [ ] **Step 1: Apply fix**

The `weak var manager` is read from a `nonisolated` context before dispatching to `@MainActor`. Since the `Task { @MainActor in ... }` closure captures `manager` at that point, we must ensure the capture happens on the main actor.

Change all delegate methods from:
```swift
nonisolated func room(_ room: LiveKit.Room, ...) {
    Task { @MainActor in
        manager?.someMethod()
    }
}
```
to use a local capture inside the `@MainActor` task rather than capturing from the nonisolated context. The minimal fix is to mark `manager` as `@MainActor`:

```swift
private final class RoomDelegateHandler: RoomDelegate, @unchecked Sendable {
    @MainActor private weak var manager: RoomManager?

    init(manager: RoomManager) {
        // Init must be called on MainActor context — RoomManager is @MainActor
        self.manager = manager
    }
    ...
```

All `Task { @MainActor in manager?.... }` accesses are then safe since they're on the main actor.

- [ ] **Step 2: Verify build**

```bash

```

If `@MainActor private weak var` causes issues with the nonisolated initializer context, the alternative minimal fix is to wrap the init assignment:
```swift
init(manager: RoomManager) {
    Task { @MainActor [weak self] in self?.manager = manager }  // Not valid for init
}
```
That won't work in an init. Use `MainActor.assumeIsolated` if the caller is always main actor:
```swift
init(manager: RoomManager) {
    MainActor.assumeIsolated { self.manager = manager }
}
```
